Implemented Features

  • Alt text for images
  • Appropriate color contrast and minimal color palette
  • Back to the top and skip to content links
  • CSS sizing using em instead of pixels
  • Each link's text being specific, not general like 'click here' and external link indicators
  • Importing only the necessary css for each page rather than the entire stylesheet
  • Labels for forms: code-side and browser-side
  • Language attributes for each page within the html
  • Lazy loading images
  • No Javascript, only css, html, and some php
  • Providing multiple file formats of each image to speed up loading times
  • Section breaks announced with a hr
  • Semantic html: a header, nav, main, and footer
  • System fonts so no loading remote fonts
  • Tables of contents on each page and post

Planned Features

  • Aria labels
  • Dark and high-contrast mode
  • Focus styles
  • Reduced motion options
  • WCAG level

Publishing A New Page

Instructions
  • Open the desired 'template.html' in VSCodium.
  • Copy the HTML template provided into a new file.
  • Save and name the file in the relevant subfolder.
  • Write the contents in Markdown and convert to HTML.
  • Update the title tag.
  • Update publication date in meta name="date"> tag and in p class="postmeta".
  • Update 'Copy & Share' link.
  • Calculate and update word count and estimated reading time in 'Statistics.'
  • Open the HTML file in a web browser with Live Server to see how it looks and test the links.
  • Proofread and fix contents.
  • Update the changelog and the word count.
  • Update the sitemap and the word count.
  • Update the homepage word count and updated date.
  • Update the sitemap.xml file.
  • Update the rss.xml file if applicable.
  • Upload all changed files to WPX using SFTP.
  • Check each is live on the site and for errors, fix as needed.
  • Commit and push all changes to the GitHub repository.

Principles

Inspired by Tracy Durnell's lovely post about her principles for how she wanted to approach her website, I'm doing the same, though, as a page instead. Consider this a living document.

Here they are in list form, then further explainations below:

  1. Undying Optimism
  2. Kindness, Always
  3. Proactive Hope
  4. Useful Beauty
  5. The Human Dream

1: Undying Optimism

This is related to the third principle, but I think it' the most important and worth highlighting on its own. Negativity has had me in a death grip for as long as I can remember. While skepticism is something I've always found useful, constant doom and gloom is... suffocating.

So just like what I try to infuse my fiction with, I want to do the same with my site.

2: Kindness, Always

Somehow we equate kindness with softness, as though it's a weakness. I'd argue it's the opposite. It's so much easier to succumb to impulses we'd rather not. It takes strength, grace, and humility to be kind to yourself and others.

While I have the latter down pat, the former... is a work in progress. This is me committing to kindness from the inside out, not merely from the outside.

3: Proactive Hope

Hope keeps us alive and moving forward. This is what I know. Though I also know it's its nothing without proactive progress. I may be only one person, though I aim to help foster connection, delight and knowledge-sharing across the internet.

4: Useful Beauty

As the textile artist and designer William Morris said: "Have nothing in your houses that you do not know to be useful or believe to be beautiful." I'd take it a step further. Why not both? Always?

In my published writing, and everything I create, I aim for this.

5: The Human Dream

We all share something in common, and that's our humanity: in all its flawed, stunning beauty. Remembering our connection to our shared existence is something I try to do every day.
